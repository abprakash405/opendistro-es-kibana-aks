

#To deploy 
kubectl apply -f elasticsearch-yml-configmap.yml

kubectl apply -f elasticsearch-statefulsets-data.yaml
kubectl apply -f elasticsearch-statefulsets-client.yaml
kubectl apply -f elasticsearch-statefulsets-ingest.yaml
#Wait for few minutes to get client,data,ingest ready before deploy master node. 
kubectl apply -f elasticsearch-statefulsets-master.yaml


#To undeploy 
kubectl delete -f elasticsearch-yml-configmap.yml
kubectl delete -f elasticsearch-statefulsets-data.yaml
kubectl delete -f elasticsearch-statefulsets-client.yaml
kubectl delete -f elasticsearch-statefulsets-ingest.yaml
kubectl delete -f elasticsearch-statefulsets-master.yaml

kubectl delete -f elasticsearch-statefulsets-master.yaml,elasticsearch-statefulsets-client.yaml,elasticsearch-statefulsets-data.yaml,elasticsearch-statefulsets-ingest.yaml


###
# As per open distro elasticsearch it will create index so mandatoring DATA node should be ready before MASTER is ready.
Otherwise it might through an error like: Open distro not initialized 
Deploy data, client, ingest and once those are ready then finally deploy master.
